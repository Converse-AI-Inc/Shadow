title: "Knowledge Base and Sources"
description: "Add links, notes, and files that Shadow can use before and during meetings."
---

The Knowledge Base lets users add reusable material that Shadow can reference across meetings.

## Supported source types

- links
- notes
- uploaded files

## What users do here

- add product or customer context before a meeting
- attach notes that should inform future conversations
- upload material the team wants Shadow to reference

## Where sources show up later

Sources are not isolated to one page. They can influence:

- meeting prep
- live meeting assistance
- review and source usage
- overlay source selection during the meeting

## Typical workflow

<Steps>
  <Step title="Open the Knowledge Base">
    Use the Sources or Knowledge Base area in the app.
  </Step>
  <Step title="Add a source">
    Choose whether the source is a link, note, or file.
  </Step>
  <Step title="Wait for processing if needed">
    Some sources may take time to process before they become fully usable.
  </Step>
  <Step title="Use those sources in prep, live, and review workflows">
    Attach or reference them in meeting-driven flows when relevant.
  </Step>
</Steps>

## Best practices

- Keep titles clear and specific.
- Remove outdated material.
- Add sources that are likely to matter repeatedly, not just once.
- Use a smaller number of high-signal sources instead of attaching everything.
