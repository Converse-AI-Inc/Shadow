The live meeting view is where Shadow moves from preparation to in-meeting assistance.

![Live answers in the moment](/images/live-answers-tough-questions.png)

## What the live view includes

- live transcript
- smart cards and nudges
- pinned approvals and clarifications
- agenda coverage
- recording controls
- transcript drawer
- overlay support for a smaller in-meeting surface

## How meetings enter the live workflow

Shadow supports more than one entry path:

- scheduled meeting opened from the dashboard
- pre-meeting calendar notification
- auto-detected supported meeting window
- manual launch through **Launch Assistant**

Supported live meeting detection covers the main desktop meeting platforms Shadow recognizes, including Microsoft Teams, Google Meet, and Zoom.

## What users should do during the call

- keep the live view or overlay available
- watch for high-signal nudges instead of reading everything
- use approvals when Shadow proposes external follow-through
- review transcript when needed for precision or recall

## Overlay workflow

The overlay is useful when the user wants Shadow visible without leaving the meeting window as the primary focus.

The overlay can stay available as the primary assistive surface during the meeting while the main window remains out of the way.

## Recording behavior

- Shadow can start from a calendar-backed meeting flow
- Shadow can start manually when the user launches it directly
- a late meeting detection can still attach the session to the detected meeting window
- when the meeting stops, the user is taken into the review workflow

## Why this page matters

This is the page where Shadow becomes operational:

- transcript accumulates in real time
- cards and nudges appear as the conversation evolves
- approvals and clarifications stay visible
- agenda coverage updates as the meeting progresses

## Live meeting best practices

- Prep before the call so the live cards are more relevant.
- Keep sources current if the user wants better context in the meeting.
- Treat approvals as controlled actions, not background automation.
- Use the overlay when the meeting app needs to stay visually dominant.
